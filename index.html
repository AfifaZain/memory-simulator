5. Android Memory Management (LMK & zRAM)

Low Memory Killer (LMK)

Unlike standard page replacement which swaps to disk, Android's LMK simply kills processes when memory is critically low. It uses an "OOM Score" (Out of Memory) to decide which app to kill first (usually background apps).

zRAM (Compressed RAM)

Instead of swapping pages to slow storage (disk/eMMC), Android uses a portion of RAM as a compressed block device. Pages are compressed before being swapped out. This is faster than disk I/O but uses CPU cycles for compression.
